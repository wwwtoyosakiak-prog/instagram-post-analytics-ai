# サイト公開手順

このアプリはNext.jsで作られています。GitHub Pagesから公開する場合は、静的サイトとして公開します。

## GitHub Pagesで公開する場合の注意

GitHub Pagesでは、画面部分とlocalStorage保存は使えます。

使える機能:

- 投稿登録
- アカウント登録
- 投稿一覧
- ダッシュボード
- レポート
- localStorage保存

GitHub Pagesは静的サイト公開のため、OpenAI API Routeやスクショから自動入力などのサーバー側APIは動きません。AI分析まで公開サイトで使いたい場合は、Vercelで公開してください。

## 1. GitHubへアップロード

このフォルダで以下を実行します。

```bash
cd "/Users/aokikousuke/Documents/Codex/2026-06-12/instagram-ai-instagram-api-instagram-csv/outputs/instagram-post-analytics-ai-full-20260614 2"
gh repo create instagram-post-analytics-ai --private --source=. --remote=origin --push
```

すでにGitHubリポジトリを作成済みの場合は、以下でアップロードします。

```bash
git push -u origin main
```

## 2. GitHub Pagesを有効化

GitHubのリポジトリ画面で以下を設定します。

1. `Settings` を開く
2. 左メニューの `Pages` を開く
3. `Build and deployment` の `Source` を `GitHub Actions` にする
4. 保存する

このフォルダには `.github/workflows/pages.yml` が入っていますが、現在は手動実行専用です。
サーバー側 API を使う本番運用は Vercel を使ってください。

## 3. GitHubへpush

```bash
git add .
git commit -m "Add GitHub Pages deployment"
git push
```

数分後、GitHubの `Actions` タブで公開処理が完了します。

## 4. 公開URL

通常は以下のURLになります。

```text
https://wwwtoyosakiak-prog.github.io/instagram-post-analytics-ai/
```

GitHubの `Settings` → `Pages` にも公開URLが表示されます。

## 5. 公開後の確認

公開URLにアクセスし、以下を確認します。

- トップページが表示される
- アカウント登録ができる
- 投稿登録ができる
- 投稿一覧から詳細ページを開ける
- サンプル分析が使える

## 6. データ保存について

初期状態ではlocalStorage保存です。

そのため、公開URLを他の人が開いても、その人のブラウザ内にだけデータが保存されます。複数人で同じデータを共有したい場合は、後から以下のどれかを追加してください。

- Supabase
- PostgreSQL
- MySQL
- Next.js API Route + データベース

## 7. 公開後に更新する方法

修正後、以下を実行します。

```bash
git add .
git commit -m "Update app"
git push
```

GitHubへpushすると、Vercelが自動で再デプロイします。
