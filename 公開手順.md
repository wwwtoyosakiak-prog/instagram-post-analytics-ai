# サイト公開手順

このアプリはNext.jsで作られているため、まずはVercelで公開するのが簡単です。

## 公開方法のおすすめ

### おすすめ: Next.jsだけをVercelで公開

以下の機能が使えます。

- 投稿登録
- アカウント登録
- 投稿一覧
- ダッシュボード
- レポート
- OpenAI API連携
- localStorage保存

注意点として、localStorage保存はブラウザごとの保存です。別のPCや別のブラウザには自動共有されません。

### PHPも使いたい場合

VercelはNext.js公開には向いていますが、このフォルダに入っているPHPサーバーをそのまま同時に動かす用途には向きません。

PHPバックエンドまで公開したい場合は、以下のどちらかを選んでください。

- PHPを使わず、まずはVercel + localStorageで公開する
- PHP対応サーバー、またはレンタルサーバーに `php-backend` を別で置く

## 1. GitHubへアップロード

このフォルダで以下を実行します。

```bash
cd "/Users/aokikousuke/Documents/Codex/2026-06-12/instagram-ai-instagram-api-instagram-csv/outputs/instagram-post-analytics-ai-full-20260614 2"
gh repo create instagram-post-analytics-ai --private --source=. --remote=origin --push
```

すでにGitHubリポジトリを作成済みの場合は、以下でアップロードします。

```bash
git push -u origin main
```

## 2. Vercelで公開

1. Vercelにログインします。
2. GitHub連携を有効にします。
3. `instagram-post-analytics-ai` リポジトリを選びます。
4. Framework Preset は `Next.js` を選びます。
5. Deploy を押します。

## 3. OpenAI APIキーを設定

Vercelのプロジェクト設定で Environment Variables を開き、以下を追加します。

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4.1-mini
```

追加後、再デプロイしてください。

## 4. 公開後の確認

公開URLにアクセスし、以下を確認します。

- トップページが表示される
- アカウント登録ができる
- 投稿登録ができる
- 設定ページでOpenAI API接続テストが成功する
- 投稿詳細ページでAI分析が動く

## 5. データ保存について

初期状態ではlocalStorage保存です。

そのため、公開URLを他の人が開いても、その人のブラウザ内にだけデータが保存されます。複数人で同じデータを共有したい場合は、後から以下のどれかを追加してください。

- Supabase
- PostgreSQL
- MySQL
- PHPバックエンド + DB

## 6. 公開後に更新する方法

修正後、以下を実行します。

```bash
git add .
git commit -m "Update app"
git push
```

GitHubへpushすると、Vercelが自動で再デプロイします。
